# Weekly Health Reset

> 日曜朝に食材が届く、週次サイクルで健康習慣を積み上げるモバイルアプリ

---

## コンセプト

「Weekly Health Reset」は、1週間を1サイクルとして食習慣を設計・実行・振り返る健康管理アプリです。
毎週日曜日の朝に食材が玄関に届き、月曜日からの7日間を計画通りに過ごせる仕組みを提供します。
食事ログは学習資産として蓄積され、週を重ねるごとにパーソナライズされた提案精度が向上します。

---

## 週次サイクル

```
月  火  水  木  金  土  日
──────────────────────────────────────────────────────
振返 振返 目標 目標 献立 カート 配達
           設定 設定 提案 連携  ↓
                              食材到着
```

### 水〜木：目標設定フェーズ

**目的：** 次週の食習慣ゴールを設定し、ベースラインを確認する。

| アクション | 詳細 |
|---|---|
| 健康目標の更新 | 体重・体脂肪・摂取カロリー目標を設定 |
| 制約・嗜好の確認 | アレルギー・苦手食材・食スタイル（低糖質など）の更新 |
| 先週の振り返り参照 | AI が先週のログを要約し改善ポイントを提示 |
| 目標カロリー配分 | 朝・昼・夕・間食のカロリー比率を設定 |

**通知:** 水曜 20:00「今週の目標を設定しましょう」

---

### 金：献立提案フェーズ

**目的：** 設定した目標に基づき、AI が7日分の献立を提案する。

| アクション | 詳細 |
|---|---|
| 7日間献立の自動生成 | 目標・制約・旬・コスト最適化を考慮したメニュー提案 |
| 献立のカスタマイズ | 各日・各食のメニューを入れ替え・調整 |
| 栄養バランス確認 | PFC バランス・微量栄養素のビジュアル表示 |
| 食材リスト生成 | 献立から必要な食材・分量を自動集計 |

**通知:** 金曜 12:00「今週の献立が提案されました」

---

### 土：カート連携フェーズ

**目的：** 確定した献立の食材を EC サービスのカートへ自動追加する。

| アクション | 詳細 |
|---|---|
| 食材リストの確認・編集 | 不要な食材の削除、数量調整、代替品の提案 |
| EC 連携カートへ追加 | 提携サービス（Oisix・Amazon Fresh など）へ一括追加 |
| 在庫チェック | 冷蔵庫の残り食材と照合して重複を排除（手動登録） |
| 注文確定 | アプリ内から注文完了、日曜配達を確認 |

**通知:** 土曜 10:00「食材の注文を完了させましょう（締切 18:00）」

---

### 日〜翌火：実行・記録フェーズ

**目的：** 食材が届いた後、献立通りに調理・食事し記録する。

| アクション | 詳細 |
|---|---|
| 食事ログ記録 | 写真・バーコード・手動でログを作成 |
| 献立との照合 | 計画通りかどうかを自動判定、逸脱時はその理由を記録 |
| リアルタイム栄養計算 | 当日の摂取カロリー・PFC の進捗表示 |
| 体重・体調メモ | 毎朝の体重・睡眠・気分を記録 |

**通知:** 毎日 20:00「今日の食事ログを記録しましょう」

---

### 月〜火：振り返りフェーズ

**目的：** 前週のデータを AI が分析し、次週の改善サイクルへつなぐ。

| アクション | 詳細 |
|---|---|
| 週次レポート生成 | 栄養達成率・目標達成度・気づきのサマリー |
| AI インサイト | 傾向分析・改善提案・次週への引き継ぎ事項 |
| ストリーク確認 | 継続週数・バッジ・達成ハイライト |
| 次週への申し送り | 気に入ったメニュー・苦手だったメニューのフラグ |

**通知:** 月曜 09:00「先週のレポートが完成しました」

## アーキテクチャ概要

```
┌─────────────────────────────────────────┐
│           Mobile App (React Native)      │
│  Expo + React Navigation + Zustand       │
└────────────────┬────────────────────────┘
                 │ REST / Realtime
┌────────────────▼────────────────────────┐
│              Supabase                    │
│  Auth │ PostgreSQL │ Storage │ Edge Fn  │
└────────────────┬────────────────────────┘
                 │
┌────────────────▼────────────────────────┐
│           AI / Integration Layer         │
│  Claude API (献立生成・分析)              │
│  EC API (Oisix / Amazon Fresh)           │
└─────────────────────────────────────────┘
```

## MVP スコープ

### In Scope（v1.0）

- [x] ユーザー認証（メール / Google OAuth）
- [x] 週次目標設定画面
- [x] AI 献立提案画面（金曜）
- [x] 食材リスト表示・手動編集
- [x] 食事ログ記録（手動入力）
- [x] 週次振り返りレポート

### Out of Scope（v2.0 以降）

- EC サービスとのリアルタイム在庫連携
- バーコードスキャンによる食材登録
- Apple Health / Google Fit 連携
- 複数人（家族）モード

## ローカル開発

```bash
# 依存関係のインストール
npm install

# 環境変数の設定
cp .env.example .env.local
# SUPABASE_URL, SUPABASE_ANON_KEY, ANTHROPIC_API_KEY を設定

# 開発サーバー起動
npx expo start

# Supabase ローカル環境
npx supabase start
npx supabase db push
```
